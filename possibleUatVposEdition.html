Provider Signature Creation

To create the provider's signature, an HTTP request needs to be made as shown in the following example:

POST https://virtualpos.services.novidea.gr/api/signdata
-h uniqueIntegratorId: 0d5d981171e44065b0e2e377feb4aacf

{
    "documentIdentifier": "7AA88EE1CDD6BEAEDC395F138957D531AB8AE7D7",
    "uniqueEntryNumber": "0",
    "signatureDatetime": "20240902142732",
    "amountPayable": "12400",
    "netValue": "2400",
    "vat": "10000",
    "totalDocumentValue": "12400",
    "posId": "87366847",
    "providerId": 1122
}

Response Example

The above request returns the following response:

{
    "signedData": "7AA88EE1CDD6BEAEDC395F138957D531AB8AE7D7;0;20240902142732;2400;10000;12400;12400;87366847",
    "signature": "MEUCIHRceJYSk1Cb6OsXDKJO/EPRxwqHzKKy9iXmxSstyozTAiEAk5SqoqWgKCCajGZqeOMwP6zm5vJzJnb5jQNHAdLmPPY=",
    "signedDataFieldsNames": "documentIdentifier;uniqueEntryNumber;signatureDatetime;netValue;vat;totalDocumentValue;amountPayable;posId"
}

The response is then used in the Deeplink Request, where the fields are filled accordingly.

Important Notes

ATTENTION!!

The fields "amountPayable": "12400", "netValue": "2400", "vat": "10000", and "totalDocumentValue": "12400" must be in integer format, with the amounts multiplied by 100.
